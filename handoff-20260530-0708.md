## Gemini — read this before testing generate with google_nano_banana

Generate defaults hero/slot providers to **`google_nano_banana`** (preview defaults to
`openai_image`). Two separate Gemini issues surfaced during local smoke testing:

1. **`output_mime_type` bug (fixed in code):** Without the fix, Gemini returns:
   `"output_mime_type parameter is only supported in Gemini Enterprise Agent Platform mode,
   not in Gemini Developer API mode."` Fix is committed in `google_nano_banana.py`.

2. **Free-tier quota exhausted (env/account, not a code bug):** After the mime fix, calls still
   failed with quota errors (`limit: 0` for `gemini-2.5-flash-preview-image`). The local dev
   script `scripts/send_request.py` sets `"default_provider_hint": "openai_image"` so local
   generate smoke tests work without Gemini. **To exercise Gemini locally:** ensure
   `GOOGLE_API_KEY` has quota/billing, then remove or change that hint in `send_request.py`
   (or pass `"google_nano_banana"` explicitly in the payload).

Prod routing is unchanged: `_resolve_provider_name` in `pack_worker.py` still prefers
`pack.default_provider_hint` → slot hint → slot-kind default (`google_nano_banana` for hero).

## Quirks / gotchas
- uvicorn runs **without `--reload`** → after any app-code edit you MUST restart it. Clean
  restart: `pgrep -f "uvicorn app.main:app" | xargs -r kill -9; sleep 2; .venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8010 > /tmp/arthor_uvicorn.log 2>&1 &`
  (Port **8010**, not 8000 — 8000 is taken by another service. Watch for multiple lingering uvicorn PIDs.)
- Each style-preview/generate call hits OpenAI (~40s, ~4 cents). The `OPENAI_API_KEY` is in `.env`.
- Generate callback POSTs to dummy `callback_url` → `ConnectError` in log is expected and non-fatal.
- DB column names: `tool_calls.run_id` (not `agent_run_id`); assets use `r2_key` (not `storage_key`).
- This repo does NOT own `agent_runs`/`tool_calls`; the bootstrap is a LOCAL stand-in only —
  never apply `db/dev/*` to a shared/prod DB.
- Cleanup when done: `docker rm -f arthor-image-pg`.
